# 使用者自訂提示詞 (User Custom Prompt)

## 【 系統名稱】
抽籤分組助手

## 【 角色】 
你是一位資深前端工程師，請幫我製作一個「純前端」的網頁小工具，使用 HTML + CSS + JavaScript（不需後端、不需框架）。

## 【 任務】
請幫我建立一個單一頁面的 Web 工具，具備「名單抽籤」與「自動分組」兩大功能。

## 【介面與使用體驗】
- 介面簡單、清楚、好操作（例如使用 Google Fonts, 圓角陰影設計）
- 不使用外部套件
- 適合在公司內部或課堂直接使用
- 行動裝置與桌機皆可正常顯示（基本 RWD）
- 介面文字：繁體中文
- 名單、抽籤與分組歷史紀錄：將名單、抽籤與分組結果儲存到 localStorage，這樣重新整理網頁後名單不會消失
- 動畫效果： 「請在抽籤時加入 2 秒的滾動動畫，增加緊張感。」
- 按鈕在無法操作時要 disabled

## 【技術限制】
- 請全部寫在單一 HTML 檔案中
- 使用原生 JavaScript（ES6）
- 程式碼需有清楚註解
- 邏輯清楚、可讀性高
- 邏輯需考慮邊界情況（例如：名單為空、抽取人數大於總人數等）
- JavaScript 將抽籤與分組邏輯拆成獨立 function
- JavaScript 避免全域變數污染
- JavaScript 提高可維護性

## 【輸出格式】
- 請直接輸出完整可執行的 HTML 原始碼
- 不要省略任何部分
- 撰寫 .gitignore、README.md 

## 【 程式碼自我審核】 
### 審核你剛寫的程式碼，檢查以下項目：
- 是否有潛在錯誤或邊界案例未處理
- 隨機性是否足夠公平
- UI/UX 是否直覺好用
- 效能與安全性（避免 XSS 等風險，例如對姓名進行適當轉義）
- 可讀性與程式碼風格 列出發現的問題（如有）並提供改進建議。若需修改，請提供修正後的完整程式碼。

## 【功能需求】
這個網站提供兩個主要功能，資料來源皆來自「使用者貼上的姓名清單」。

### 一、名單輸入
- 提供一個大型 textarea
- 使用者可貼上多個姓名
- 一行一個姓名
- 自動忽略空白行
- 去除前後空白字元
- 內部以陣列形式儲存名單
- 發現有重複的姓名，在預覽名單處標記出來，並提供一個按鈕讓使用者一次性移除重複的姓名
- 提供「模擬名單」功能，讓使用者可快速匯入一批名單
- 提供「重置名單」按鈕，可一次刪除所有名單功能，刪除前讓使用者確認

### 二、功能一：名單抽籤
- 隨機抽取姓名
- 可設定「抽取人數」（例如一次抽 1 人、3 人）
- 可設定是否「允許重複抽取」
- 抽籤結果要即時顯示在畫面上
- 清楚顯示本次抽中的姓名
- 若名單不足時要有提示訊息
- 提供「重新抽籤」功能，一次清除抽籤結果，刪除前讓使用者確認
- 顯示目前剩餘人數
- 抽籤結果加入動畫或高亮效果 

### 三、功能二：自動分組
- 可設定「每組幾個人」
- 依據目前名單隨機分組
- 所有人都要被分到組別
- 最後一組人數不足時仍可成立一組
- 分組結果需以「視覺化卡片或區塊」方式顯示
  - 每一組一個區塊
  - 顯示第幾組
  - 顯示該組成員姓名清單
- 分組完成後，提供下載 CSV 檔案功能
- 提供「重新分組」功能，重新分組前讓使用者確認
- 分組區塊使用不同背景色區分
